# MediTrack — Backend Schema Document

**Version:** 1.0 | **Date:** June 2026

---

## 1. Database Overview

MediTrack uses PostgreSQL hosted on Supabase. All tables use UUID primary keys. Row-Level Security (RLS) is enabled on all user-data tables. The schema is designed to support both authenticated users and guest sessions.

> **Naming Convention:** snake_case for all table and column names. Tables named as plural nouns. Foreign keys named as `{referenced_table_singular}_id`.

---

## 2. Table Definitions

### 2.1 users (managed by Supabase Auth)

This table is auto-managed by Supabase. We extend it with a `profiles` table.

| Column | Type | Notes |
|---|---|---|
| id | UUID | Primary key — matches auth.users.id |
| email | TEXT UNIQUE NOT NULL | User's email |
| created_at | TIMESTAMPTZ DEFAULT now() | Account creation time |
| last_sign_in_at | TIMESTAMPTZ | Last login timestamp |

### 2.2 profiles

Extended user information for AI personalization.

| Column | Type | Notes |
|---|---|---|
| id | UUID PRIMARY KEY | References auth.users(id) ON DELETE CASCADE |
| full_name | TEXT | User's display name |
| age | INTEGER CHECK (age >= 1 AND age <= 120) | User age — improves AI accuracy |
| sex | TEXT CHECK (sex IN ('male','female','prefer_not_to_say')) | Biological sex for medical context |
| pre_conditions | TEXT[] DEFAULT '{}' | e.g. '{"diabetes","hypertension"}' |
| created_at | TIMESTAMPTZ DEFAULT now() | |
| updated_at | TIMESTAMPTZ DEFAULT now() | Auto-updated via trigger |

### 2.3 sessions

Each completed symptom analysis creates one session record.

| Column | Type | Notes |
|---|---|---|
| id | UUID PRIMARY KEY DEFAULT gen_random_uuid() | Session identifier |
| user_id | UUID REFERENCES auth.users(id) ON DELETE CASCADE | NULL for guest sessions |
| guest_token | TEXT | Random token for guest session identification |
| created_at | TIMESTAMPTZ DEFAULT now() | When analysis was run |
| urgency_level | TEXT NOT NULL | EMERGENCY \| SEE_DOCTOR_TODAY \| MONITOR \| ROUTINE |
| urgency_color | TEXT NOT NULL | red \| orange \| yellow \| green |
| top_condition | TEXT NOT NULL | Name of the highest-confidence condition |
| raw_ai_response | JSONB NOT NULL | Full AI response stored for history display |
| symptoms_summary | JSONB NOT NULL | Array of { region, symptom, severity, duration } |
| is_offline | BOOLEAN DEFAULT false | True if generated by local fallback engine |
| pdf_url | TEXT | Supabase Storage URL if user exported PDF |

### 2.4 symptoms (session-level detail)

Normalized symptom rows for analytics and future ML training data.

| Column | Type | Notes |
|---|---|---|
| id | UUID PRIMARY KEY DEFAULT gen_random_uuid() | |
| session_id | UUID REFERENCES sessions(id) ON DELETE CASCADE | Parent session |
| body_region | TEXT NOT NULL | e.g. 'chest', 'abdomen', 'head' |
| symptom_name | TEXT NOT NULL | e.g. 'sharp pain', 'nausea' |
| severity | INTEGER CHECK (severity >= 1 AND severity <= 10) | 1-10 scale |
| duration_label | TEXT | e.g. '1-3 days', 'just started' |
| free_text | TEXT | Optional user description (max 500 chars) |

### 2.5 conditions (per session)

The AI-returned conditions associated with a session.

| Column | Type | Notes |
|---|---|---|
| id | UUID PRIMARY KEY DEFAULT gen_random_uuid() | |
| session_id | UUID REFERENCES sessions(id) ON DELETE CASCADE | Parent session |
| condition_name | TEXT NOT NULL | e.g. 'Pleuritis' |
| confidence_pct | INTEGER CHECK (confidence_pct >= 0 AND confidence_pct <= 100) | AI confidence |
| description | TEXT NOT NULL | 2-sentence plain-English description |
| learn_more_url | TEXT | Link to Mayo Clinic / WHO page |
| rank | INTEGER | 1 = top condition, 2 = second, 3 = third |

### 2.6 offline_sessions (local SQLite — client only)

SQLite schema for offline-first local storage. Same shape as server sessions for easy sync.

```sql
CREATE TABLE offline_sessions (
  id TEXT PRIMARY KEY,
  user_id TEXT,
  created_at TEXT,
  urgency_level TEXT,
  top_condition TEXT,
  raw_result TEXT,        -- JSON stringified
  symptoms TEXT,          -- JSON stringified
  is_offline INTEGER DEFAULT 0,
  synced INTEGER DEFAULT 0  -- 0 = not synced, 1 = synced to server
);
```

## 3. Row-Level Security Policies

| Table | Policy | Rule |
|---|---|---|
| profiles | SELECT own profile | auth.uid() = id |
| profiles | UPDATE own profile | auth.uid() = id |
| sessions | SELECT own sessions | auth.uid() = user_id |
| sessions | INSERT own sessions | auth.uid() = user_id |
| sessions | DELETE own sessions | auth.uid() = user_id |
| symptoms | SELECT via session | session_id IN (SELECT id FROM sessions WHERE user_id = auth.uid()) |
| conditions | SELECT via session | session_id IN (SELECT id FROM sessions WHERE user_id = auth.uid()) |

## 4. Key Indexes

```sql
CREATE INDEX idx_sessions_user_id ON sessions(user_id);
CREATE INDEX idx_sessions_created_at ON sessions(created_at DESC);
CREATE INDEX idx_symptoms_session_id ON symptoms(session_id);
CREATE INDEX idx_conditions_session_id ON conditions(session_id);
```

---

## 5. Sync Logic (Local → Server)

1. On app launch with network: check `offline_sessions WHERE synced = 0`
2. For each unsynced session: POST to `/api/sessions/sync` with full payload
3. On 200 OK: mark SQLite record `synced = 1`
4. If server returns conflict (duplicate id): server version wins, local updated
5. Guest sessions: never synced unless user creates account
